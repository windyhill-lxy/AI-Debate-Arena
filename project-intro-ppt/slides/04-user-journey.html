User Journey 一次训练如何发生
使用路径

从首页建房开始，系统把复杂辩论拆成可执行的训练流程。

1. 建立房间输入辩题，选择正式赛制、计时方式、可见性模式和参与身份。
→
2. 导入资料用户上传参考材料，系统建立可引用的知识来源，供 RAG 检索使用。
→
3. 自动推进赛程状态机控制当前环节，AI 辩手按角色职责发言，WebSocket 实时同步。
→
4. 人类介入轮到用户时暂停等待输入；AI 教练给思路与资料编号，但不替用户自动发言。
→
5. 复盘输出裁判生成胜负、主战场、失误与改进建议，逐字稿可导出和分享。

这条路径的关键是：学生不只看到 AI 的答案，还能看到“答案是如何被检索、判断、核查并纳入赛程”的过程。
