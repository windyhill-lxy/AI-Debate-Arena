# AI 辩论场 · 桌面程序



本目录为**独立打包入口**，不会修改项目根目录下的源码与脚本。



## 快速开始（开发调试）



1. 确保项目根目录已运行过 `bootstrap.bat`（存在 `tools\python` 与 `tools\node`）。

2. 首次使用：双击 **`准备程序.bat`**（构建前端并安装 Electron）。

3. 日常启动：双击 **`启动 AI辩论.bat`**（唯一入口，无需 `start.bat` / `start-lan.bat`）。



## 打包成 exe + 依赖（分发给他人）



1. 确保已运行 `bootstrap.bat`，且 `tools\python` 已安装后端依赖。

2. 双击 **`打包发布.bat`**（约 5–15 分钟，含复制 Python ~470MB）。

3. 输出目录：**`release/AI辩论场/`**



```

release/AI辩论场/

  AI辩论场.exe              ← 双击启动

  .env.example              ← 复制为 .env 并填入 API 密钥

  使用说明.txt

  resources/app-core/       ← 内置 Python、后端、前端（勿删）

  *.dll / *.pak ...         ← Electron 运行依赖

```



整个 **`AI辩论场`** 文件夹可复制到其他 Windows 电脑直接运行，**无需安装 Python / Node**。



## 特性



- **界面与网页端一致**：使用同一套 `frontend` 构建产物。

- **联机内置**：后端与前端均监听 `0.0.0.0`，本机与局域网均可联机。

- **单窗口应用**：Electron 壳加载 `http://127.0.0.1:5173`，菜单可一键复制联机地址。

- **原项目不动**：所有打包脚本与资源仅在本目录。



## 目录结构



```

应用程序/

  启动 AI辩论.bat      ← 开发/本机调试

  打包发布.bat          ← 生成 exe + 依赖发行包

  准备程序.bat          ← 首次或更新 UI 后运行

  scripts/              ← 打包专用脚本

  electron/             ← 桌面壳

  release/AI辩论场/     ← 打包输出（运行打包后生成）

  assets/frontend-dist/ ← 构建后的前端

```



## 联机说明



主持方在本机创建联机房间后，通过菜单 **文件 → 复制联机地址（局域网）** 将地址发给同学。  

同学用浏览器打开该地址即可加入（无需安装本程序）。



防火墙需放行端口 **5173**（页面）与 **9000**（API）。



## 故障排除



| 现象 | 处理 |

|------|------|

| 提示缺少 Python/Node | 运行项目根目录 `bootstrap.bat` |

| 提示缺少前端资源 | 运行 `准备程序.bat` |

| 端口占用 | 关闭其他辩论程序窗口，或重启电脑 |

| UI 更新后仍是旧版 | 重新运行 `准备程序.bat` 或 `打包发布.bat` |

| 打包失败 / 目录被占用 | 关闭「启动 AI辩论.bat」窗口后重试 |
